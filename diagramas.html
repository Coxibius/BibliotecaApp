BibliotecaApp — Documentación UML

Arquitectura del sistema documentada con 5 diagramas UML. Cada diagrama incluye su explicación y leyenda.

Diagrama de Casos de Uso
¿Qué es?

Un diagrama de casos de uso muestra qué puede hacer el sistema y quién lo usa. No describe cómo lo hace, solo las funciones disponibles y los actores involucrados.

Actores del sistema
  • Bibliotecario — Administrador del sistema. Gestiona autores, libros, estudiantes y todos los préstamos.
  • Estudiante — Usuario final. Puede ver el catálogo de libros disponibles y solicitar un préstamo.
Casos de uso y su significado
Caso de Uso ¿Quién? Descripción
Gestionar Autores Bibliotecario CRUD completo de autores (crear, editar, eliminar, ver).
Gestionar Libros Bibliotecario CRUD completo de libros, asignando un autor a cada uno.
Gestionar Estudiantes Bibliotecario CRUD completo de estudiantes registrados.
Registrar Préstamo Ambos Asocia un libro disponible a un estudiante. El libro pasa a no disponible.
Registrar Devolución Bibliotecario Marca el préstamo como devuelto y libera el libro.
Eliminar Préstamo Bibliotecario Elimina el registro y restaura la disponibilidad del libro si no fue devuelto.
Ver Catálogo Estudiante Página de inicio pública que muestra todos los libros disponibles. 

      


      
      

        
Diagrama de Clases


        

          
 ¿Qué es?

          
Un diagrama de clases representa la estructura estática del sistema: las
            entidades (clases/modelos), sus atributos con tipos de datos, y las relaciones entre ellas. Es la base del
            modelo de base de datos.

        


        

          
 Relaciones y multiplicidad

          
    
            
  • Autor → Libro (1 a muchos): Un autor puede haber escrito varios libros, pero cada libro
              tiene un único autor (AutorId).
    • 
            
  • Libro → Prestamo (1 a muchos): Un libro puede estar en varios préstamos a lo largo del
              tiempo, pero solo uno activo (Disponible = false).
    • 
            
  • Estudiante → Prestamo (1 a muchos): Un estudiante puede tener varios préstamos
              registrados.
    • 
          

        


        

          
 Atributos destacados

          

            
              

                Clase
                Atributo clave
                Notas
              

            
            
              

                Libro
                Disponible: bool
                Controla si el libro puede prestarse. Por defecto true.
              

              

                Prestamo
                FechaDevolucion: DateTime?
                Es nullable (?): si es null, el préstamo sigue activo.
              

              

                Libro / Autor
                string?
                Nullable para evitar errores de validación en MVC con Nullable enable.
              

            
          

        


        

        
        

      


      
      

        
Diagrama de Secuencia


        

          
 ¿Qué es?

          
Un diagrama de secuencia muestra la interacción entre componentes a lo largo del
              tiempo, de arriba hacia abajo. Las flechas representan mensajes o llamadas entre participantes.

        


        

          
 Flujo representado: Crear un Préstamo

          
Este diagrama traza el camino completo de registrar un préstamo exitoso:

          
    
            
  1. El Usuario llena el formulario en la Vista Create y presiona Guardar.
            
    2. 
            
  2. La Vista envía un POST HTTP al PrestamosController.
    3. 
            
  3. El controlador valida el modelo (ModelState.IsValid).
    4. 
            
  4. Busca el libro seleccionado en la BD para cambiar Disponible = false.
    5. 
            
  5. Guarda el nuevo préstamo con SaveChangesAsync().
    6. 
            
  6. Redirige al índice de préstamos con la lista actualizada.
    7.

Diagrama de Estados


        

          
 ¿Qué es?

          
Un diagrama de estados (o de máquina de estados) muestra todos los estados
              posibles de un objeto y los eventos o acciones que lo hacen cambiar de un estado a otro.

        


        

          
 Objeto representado: la propiedad Disponible de un Libro

          
El campo Disponible (bool) en la clase Libro define en cuál de estos estados se
            encuentra:

          

            
              

                Estado
                Valor
                Significado
              

            
            
              

                Disponible
                true
                El libro puede ser prestado. Aparece en el catálogo.
              

              

                Prestado
                false
                El libro está en manos de un estudiante. No aparece en el catálogo.
              

            
          

        


        

          
 Transiciones

          
    
            
  • Libro creado → nace como Disponible.
    • 
            
  • Disponible → Prestado: cuando el bibliotecario registra un nuevo préstamo
              (Disponible = false).
    • 
            
  • Prestado → Disponible: cuando se registra la devolución
              (Disponible = true) o se elimina el préstamo activo.
    • 
            
  • Disponible → Fin: cuando el libro se elimina del sistema.
    • 
          

        


        

        
        

      


      
      

        
Diagrama de Actividades


        

          
 ¿Qué es?

          
Un diagrama de actividades es similar a un diagrama de flujo. Representa el algoritmo
              o lógica interna de un proceso, mostrando decisiones (if/else), acciones secuenciales y
            puntos de inicio/fin.

        


        

          
 Proceso representado: método Devolver(int id)

          
Este diagrama mapea exactamente el código del método Devolver en
            PrestamosController.cs:

          
    
            
  1. Recibe el id del préstamo a devolver.
    2. 
            
  2. Busca el préstamo en la BD con Include(Libro) para cargar el libro relacionado.
    3. 
            
  3. Decisión 1: Si el préstamo no existe → retorna 404 Not Found.
    4. 
            
  4. Decisión 2: Si FechaDevolucion ya tiene valor → el préstamo ya fue
              devuelto, no hace nada.
    5. 
            
  5. Si sigue activo: asigna FechaDevolucion = DateTime.Now.
    6. 
            
  6. Decisión 3: Si el libro relacionado existe → cambia Disponible = true y lo
              actualiza.
    7. 
            
  7. Actualiza el préstamo y guarda con SaveChangesAsync().
    8. 
            
  8. Redirige al índice de préstamos.
    9.
